# 💸 돈어디감 (Where Did My Money Go?)

> 분명 있었는데… 어디로 갔을까? <br>
> 수입과 지출을 기록하고 돈의 흐름을 한눈에 확인하는 수입/지출 관리 웹 앱입니다.

---

## 📌 프로젝트 소개

**돈어디감**은 수입과 지출을 기록하고, 월별 통계와 잔액을 확인할 수 있는 **간단한 가계부 웹 애플리케이션**입니다.

사용자는 다음과 같은 기능을 통해 자신의 소비 흐름을 관리할 수 있습니다.

- 수입 / 지출 기록
- 거래 내역 조회 및 수정
- 월별 통계 확인
- 필터 및 검색 기능
- 로컬 저장을 통한 사용자 설정 유지

---

## 🧰 사용 기술

| 기술              | 설명             |
| ----------------- | ---------------- |
| HTML              | 페이지 구조      |

## ⚙️ 주요 기능

### 1️⃣ 수입 / 지출 기록

- 수입과 지출을 구분하여 기록
- 날짜, 카테고리, 금액, 설명 입력
- 수정 및 삭제 가능

---

### 2️⃣ 거래 내역 관리 (CRUD)

| 기능   | 설명                  |
| ------ | --------------------- |
| Create | 새로운 거래 내역 추가 |
| Read   | 전체 거래 내역 조회   |
| Update | 거래 내역 수정        |
| Delete | 거래 내역 삭제        |

---

### 3️⃣ 선택 삭제

- 체크박스를 이용한 **다중 선택**
- `Promise.all()`을 활용한 **병렬 삭제 처리**

---

### 4️⃣ 필터 / 정렬 / 검색

#### 필터

- 수입 / 지출 필터
- 카테고리 필터

#### 정렬

- 날짜순
- 금액순
- 오름차순 / 내림차순

#### 검색

- 설명(description) 기반 검색
- debounce 적용으로 검색 성능 최적화

---

### 5️⃣ 월별 통계

`reduce()`와 `Date` 객체를 활용하여:

- 월별 수입
- 월별 지출
- 월별 잔액

을 계산하여 표시합니다.

---

### 6️⃣ 사용자 설정 저장

#### localStorage

다음 설정을 저장합니다.

- 필터 상태
- 정렬 기준
- 정렬 방향

페이지를 다시 열어도 이전 설정이 유지됩니다.

---

### 7️⃣ 폼 임시 저장

`sessionStorage`를 활용하여

- 작성 중인 폼 데이터 자동 저장
- 새로고침 시 데이터 복원

을 지원합니다.

---

### 8️⃣ 로딩 상태 표시

서버 요청 중에는 **로딩 UI**를 표시하여 사용자 경험을 개선했습니다.

---

### 9️⃣ 에러 처리

모든 API 요청에 대해

- try / catch
- 사용자 친화적 에러 메시지

를 적용했습니다.

## 📚프로젝트 회고 / 느낀점

이번 프로젝트를 통해 단순히 기능을 구현하는 것뿐 아니라, **사용자가 실제로 사용할 수 있는 형태의 웹 애플리케이션을 만드는 경험**을 할 수 있었습니다. 처음에는 거래 기록을 추가하고 삭제하는 기본적인 CRUD 기능을 구현하는 것에 집중했지만, 프로젝트를 진행하면서 점점 **사용성을 고려한 기능과 UI 개선의 중요성**을 느끼게 되었습니다.

특히 필터, 정렬, 검색 기능을 구현하면서 단순히 데이터를 보여주는 것보다 **사용자가 원하는 정보를 쉽게 찾을 수 있도록 구조를 설계하는 과정**이 중요하다는 것을 배울 수 있었습니다. 또한 소비 데이터를 기반으로 **월별 통계와 소비 분석을 시각적으로 표현하는 기능을 추가하면서 데이터 처리와 UI 연결 과정에 대한 이해도도 높아졌습니다.**

UI 측면에서는 카드 형태의 대시보드, 거래 리스트, 필터 영역 등을 구성하며 **가독성과 사용 흐름을 고려한 화면 구조 설계**를 고민하게 되었고, 다크모드와 같은 사용자 편의 기능을 구현하면서 **CSS 변수와 상태 관리 방식**에 대해서도 학습할 수 있었습니다.

또한 개발 과정에서 코드 구조를 정리하고 기능별로 파일을 분리하면서 **유지보수와 확장성을 고려한 코드 설계의 필요성**도 느낄 수 있었습니다. 작은 기능을 추가할 때에도 전체 구조에 어떤 영향을 미치는지 고민하는 과정이 인상적이었습니다.

이번 프로젝트를 통해 **프론트엔드에서 데이터를 관리하고 사용자 인터페이스와 연결하는 흐름**을 전체적으로 경험할 수 있었으며, 앞으로는 로그인 기반 데이터 관리나 차트 시각화 기능 등을 추가하여 **실제 서비스 형태에 가까운 프로젝트로 확장해 보고 싶습니다.**
